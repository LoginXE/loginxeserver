로그인XE 서버 모듈 설치 방법

로그인XE 서버 모듈은 설치 이후 바로 사용할 수 없으며, 반드시 설정 과정을 거쳐야 합니다. 설치한 LoginXE를 어떻게 각 로그인 서비스와 연결하는지 설명드리겠습니다.

네이버

우선 네아로 개발자 센터로 접속합니다. 네이버 아이디로 로그인하신 후, 새 애플리케이션 등록 링크를 클릭해주세요.

등록 페이지가 열리면, 상단에 기초 정보를 입력합니다.

서비스 환경에 4가지 항목이 있는데, 그중에서 LoginXE 연동에 필요한 것은 Web입니다.

서비스 URL엔 사이트 주소를, Callback url엔 로그인XE 서버 모듈의 Callback url(모듈 설정 페이지의 입력란을 클릭하여 전부 선택을 한 뒤 Ctrl+C로 복사가 가능합니다)를 붙여넣어 줍니다. Callback url 등록이 완료되면 + 를 눌러 등록을 해주세요.(+를 누르지 않으면 등록이 되지 않습니다)

그뒤, 담당자 연락처 밑의 휴대폰 수정/인증을 클릭하여 담당자의 휴대폰 번호를 등록해 주셔야 합니다.(휴대폰 등록 없이 진행 불가능)

약관 동의를 하신 뒤 확인 버튼을 눌러 주시면 등록이 완료됩니다.

마지막으로, 일반-API 상태에서 개발중 상태인 것을 서비스 적용을 클릭하여 상태 변경후, 확인 클릭으로 저장하시면 됩니다.

등록 완료후, 애플리케이션 개요에서 Client ID와, Client Secret(보기 버튼을 눌러서 볼 수 있으며 재발급 버튼으로 Client Secret를 바꿀 수 있습니다) 를 복사해서 모듈 설정에 붙여넣어 주시면 됩니다.

Github

  1. https://github.com/settings/applications/new 에서 새 앱을 등록합니다. Application name:인증시 보일 이름입니다. Homepage URL:홈페이지 주소입니다. 로그인XE 서버 모듈이 깔린 홈페이지 주소로 해두는 것이 좋습니다. Application description:인증화면에서 보일 서비스에 대한 설명입니다. Authorization callback URL:관리자 화면의 Github Callback url 의 값을 입력해 주세요.

  2. 등록이후 나오는 ClientID와 ClientSecret를 관리자 화면의 Github ClientID/ClientSecret에 입력해 주고 저장해 줍니다.

LoginXE 서버 모듈 설정

  • ClientID 및 Clientkey는 각 API 발급 과정에서 받으실 수 있습니다.
  • 로그인XE 사용 허가 도메인에 반드시 사용할 도메인을 http://나 주소 뒤 / 없이, 도메인만 입력해주세요. 도메인 구분은 줄바꿈으로 합니다.
  • Callback url 입력란과 클라이언트 아이디/key 입력란은 클릭으로 전체 선택이 가능합니다.

설정을 마치신 후 저장 버튼을 눌러 저장하시면 됩니다.
